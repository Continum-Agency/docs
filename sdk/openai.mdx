## Overview

The Continum SDK provides seamless integration with OpenAI's GPT models, including GPT-4o, GPT-5, and reasoning models like o1 and o3.

## Supported Models

| Model Name | API Model ID | Capabilities |
|------------|--------------|--------------|
| `gpt_5` | `gpt-5` | Latest flagship model |
| `gpt_4o` | `gpt-4o` | Multimodal (text + vision) |
| `gpt_4_turbo` | `gpt-4-turbo` | Fast variant |
| `o3` | `o3` | Advanced reasoning |
| `o3_mini` | `o3-mini` | Efficient reasoning |
| `o1` | `o1` | Reasoning model |

## Basic Usage

```typescript
import { Continum } from '@continum/sdk';

const continum = new Continum({
  continumKey: process.env.CONTINUM_KEY,
  apiKeys: {
    openai: process.env.OPENAI_API_KEY
  },
  defaultSandbox: 'your-sandbox-slug'
});

const response = await continum.llm.openai.gpt_4o.chat({
  messages: [
    { role: 'system', content: 'You are a helpful assistant.' },
    { role: 'user', content: 'What is the capital of France?' }
  ]
});

console.log(response.content);
```

## Reasoning Models

OpenAI's o1, o3, and o3-mini models support extended reasoning:

```typescript
const response = await continum.llm.openai.o3.chat({
  messages: [
    { role: 'user', content: 'Solve this complex problem step by step...' }
  ],
  reasoning_effort: 'high'  // Options: 'low', 'medium', 'high'
});

// Access the reasoning trace
console.log('Thinking:', response.thinkingBlock);
console.log('Answer:', response.content);
```

## Vision Support

GPT-4o and GPT-5 support image inputs:

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [{
    role: 'user',
    content: [
      { type: 'text', text: 'What is in this image?' },
      { 
        type: 'image_url', 
        image_url: {
          url: 'https://example.com/image.jpg',
          detail: 'high'  // Options: 'low', 'high', 'auto'
        }
      }
    ]
  }]
});
```

## Function Calling

Use tools/function calling with OpenAI models:

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [
    { role: 'user', content: 'What is the weather in San Francisco?' }
  ],
  tools: [
    {
      type: 'function',
      function: {
        name: 'get_weather',
        description: 'Get the current weather in a location',
        parameters: {
          type: 'object',
          properties: {
            location: {
              type: 'string',
              description: 'The city and state, e.g. San Francisco, CA'
            },
            unit: {
              type: 'string',
              enum: ['celsius', 'fahrenheit']
            }
          },
          required: ['location']
        }
      }
    }
  ],
  tool_choice: 'auto'
});

// Check if model wants to call a function
if (response.raw.choices[0].message.tool_calls) {
  const toolCall = response.raw.choices[0].message.tool_calls[0];
  console.log('Function:', toolCall.function.name);
  console.log('Arguments:', toolCall.function.arguments);
}
```

## Streaming Responses

Stream responses for real-time output:

```typescript
const stream = await continum.llm.openai.gpt_4o.chat({
  messages: [
    { role: 'user', content: 'Write a long story about a robot.' }
  ],
  stream: true
});

for await (const chunk of stream) {
  process.stdout.write(chunk.content);
}
```

## Advanced Options

### Temperature Control

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [{ role: 'user', content: 'Be creative!' }],
  temperature: 0.9,  // 0.0 to 2.0 (higher = more creative)
  top_p: 0.95
});
```

### Max Tokens

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [{ role: 'user', content: 'Summarize briefly.' }],
  maxTokens: 100  // Limit response length
});
```

### Response Format

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [{ role: 'user', content: 'Return JSON with name and age.' }],
  response_format: { type: 'json_object' }
});

const data = JSON.parse(response.content);
```

## Token Usage

Track token consumption:

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [{ role: 'user', content: 'Hello!' }]
});

console.log('Input tokens:', response.promptTokens);
console.log('Output tokens:', response.outputTokens);
console.log('Total tokens:', response.promptTokens + response.outputTokens);
```

## Error Handling

```typescript
try {
  const response = await continum.llm.openai.gpt_4o.chat({
    messages: [{ role: 'user', content: 'Hello' }]
  });
} catch (error) {
  if (error.message.includes('rate_limit')) {
    console.error('Rate limit exceeded');
  } else if (error.message.includes('invalid_api_key')) {
    console.error('Invalid OpenAI API key');
  } else {
    console.error('Error:', error.message);
  }
}
```

## Best Practices

### 1. Use Appropriate Models

```typescript
// For simple tasks - use GPT-4o (fast and cheap)
const simple = await continum.llm.openai.gpt_4o.chat({
  messages: [{ role: 'user', content: 'What is 2+2?' }]
});

// For complex reasoning - use o3
const complex = await continum.llm.openai.o3.chat({
  messages: [{ role: 'user', content: 'Solve this complex math problem...' }],
  reasoning_effort: 'high'
});
```

### 2. Set Reasonable Token Limits

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [{ role: 'user', content: 'Summarize this article...' }],
  maxTokens: 500  // Prevent runaway costs
});
```

### 3. Use System Prompts

```typescript
const response = await continum.llm.openai.gpt_4o.chat({
  messages: [
    { 
      role: 'system', 
      content: 'You are a helpful assistant that always responds in JSON format.'
    },
    { role: 'user', content: 'Tell me about Paris.' }
  ]
});
```
