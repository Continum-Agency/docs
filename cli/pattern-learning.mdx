## Overview

Continum CLI uses a combination of built-in patterns and AI-powered detection to catch credentials. When it finds something that looks like a credential but doesn't match any known pattern, it can learn from your approval and share that knowledge with your team.

## How Pattern Detection Works

### Built-in Patterns

The CLI ships with regex patterns for common credential types:

```regex
# AWS Access Key
AKIA[0-9A-Z]{16}

# Stripe Live Key
sk_live_[0-9a-zA-Z]{24,}

# GitHub Personal Access Token
ghp_[0-9a-zA-Z]{36}
```

These patterns catch known credential formats instantly with zero false positives.

### Unknown Pattern Detection

When the CLI encounters a high-entropy string in a suspicious context, it flags it as a potential credential:

<Steps>
  <Step title="Entropy Analysis">
    The CLI calculates the randomness of strings. High entropy suggests a generated credential.
  </Step>
  
  <Step title="Context Analysis">
    Variable names, comments, and surrounding code provide context. Strings assigned to variables like `apiKey`, `password`, or `token` are flagged.
  </Step>
  
  <Step title="Pattern Extraction">
    The CLI extracts a regex pattern from the detected string. For example, `acme_prod_x7k9m2p8q4w6` becomes `acme_prod_[a-z0-9]{16}`.
  </Step>
  
  <Step title="User Approval">
    You're prompted to approve, ignore, or block the pattern.
  </Step>
</Steps>

## Pattern Approval Flow

When an unknown pattern is detected, you see this prompt:

```bash
⚠️  POSSIBLE CREDENTIAL DETECTED

config.ts (line 14)
──────────────────────────────────────────────────────
Type:    UNKNOWN_PATTERN (HIGH confidence)
Found:   acme_prod_x7k9••••••••
Pattern: acme_prod_[a-z0-9]{16}

This looks like a credential, but it's not in our pattern library.

Options:
  [b] Block this commit
  [a] Approve pattern and block (will catch in future)
  [i] Ignore this pattern
  [c] Continue anyway (not recommended)

Choice:
```

### Option: Block This Commit

```bash
Choice: b
```

- Blocks the current commit
- Does NOT save the pattern
- You'll be prompted again if you encounter this pattern in the future

**Use when:** You need to remove the credential but don't want to add it to your pattern library yet.

### Option: Approve Pattern and Block

```bash
Choice: a

Pattern description: ACME Production API Key
Severity level: HIGH

✓ Pattern saved to your library
✓ This pattern will now be caught locally on future commits

Commit blocked. Remove the credential and try again.
```

- Blocks the current commit
- Saves the pattern to your Continum account
- Syncs to all team members automatically
- Future commits with this pattern are caught instantly

**Use when:** This is a real credential format your team should always catch.

### Option: Ignore This Pattern

```bash
Choice: i

✓ Pattern ignored
```

- Allows the current commit
- Adds the pattern to your ignore list
- You won't be prompted for this pattern again

**Use when:** This is a false positive (test data, example code, etc.)

### Option: Continue Anyway

```bash
Choice: c

⚠️  Continuing without blocking (not recommended)
```

- Allows the current commit
- Does NOT save or ignore the pattern
- You'll be prompted again in the future

**Use when:** You're certain this isn't a credential and need to commit urgently. Not recommended for regular use.

## Pattern Syncing

When you approve a pattern, it's saved to your Continum account and synced to your team:

<Steps>
  <Step title="Pattern Saved">
    The pattern is saved to the Continum platform with your description and severity level.
  </Step>
  
  <Step title="Team Notification">
    Other team members are notified of the new pattern (optional dashboard notification).
  </Step>
  
  <Step title="Automatic Sync">
    The next time a team member runs `continum scan`, the pattern is downloaded automatically.
  </Step>
  
  <Step title="Local Caching">
    Patterns are cached locally for offline use. They're refreshed periodically.
  </Step>
</Steps>

### Manual Pattern Update

Force an immediate pattern update:

```bash
continum patterns update
```

This downloads the latest patterns from your team without waiting for the next scan.

## Confidence Levels

The CLI assigns confidence levels to unknown patterns:

<Tabs>
  <Tab title="HIGH Confidence">
    **Characteristics:**
    - High entropy (very random)
    - Suspicious variable name (`apiKey`, `secret`, `token`)
    - Consistent format (looks like a generated key)
    
    **Example:**
    ```javascript
    const apiKey = "acme_prod_x7k9m2p8q4w6";
    ```
    
    **Recommendation:** Approve if it's a real credential format
  </Tab>
  
  <Tab title="MEDIUM Confidence">
    **Characteristics:**
    - Moderate entropy
    - Somewhat suspicious context
    - Could be a credential or random data
    
    **Example:**
    ```javascript
    const identifier = "user_abc123def456";
    ```
    
    **Recommendation:** Review carefully before approving
  </Tab>
  
  <Tab title="LOW Confidence">
    **Characteristics:**
    - Lower entropy
    - Less suspicious context
    - Likely not a credential
    
    **Example:**
    ```javascript
    const code = "ABC123";
    ```
    
    **Recommendation:** Usually safe to ignore
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Too many false positives">
    - Add test files to your `ignore` list in `.continum.json`
    - Use the "ignore" option for patterns that aren't credentials
    - Adjust your `block` and `warn` severity levels
  </Accordion>
  
  <Accordion title="Pattern not syncing to team">
    - Ensure team members run `continum patterns update`
    - Check that everyone is using the same Continum account/organization
    - Verify network connectivity to Continum API
  </Accordion>
  
  <Accordion title="Can't approve patterns">
    - Ensure you're logged in: `continum status`
    - Check your API key has the necessary permissions
    - Verify you're not in `--strict` mode
  </Accordion>
</AccordionGroup>
