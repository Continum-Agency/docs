## Quick Start

Get up and running with Continum CLI in three simple steps:

<Steps>
  <Step title="Install the CLI">
    Install Continum CLI globally on your machine:
    
    ```bash
    npm install -g @continum/cli
    ```
    
    This makes the `continum` command available system-wide.
  </Step>
  
  <Step title="Login to Continum">
    Authenticate with your Continum account:
    
    ```bash
    continum login
    ```
    
    This opens your browser for secure authentication. Sign in once and you're ready to go.
    
    <Note>
      Your API key will be displayed in both the browser and terminal. Store it securely for your records.
    </Note>
  </Step>
  
  <Step title="Initialize in Your Project">
    Navigate to your project and initialize Continum:
    
    ```bash
    cd your-project
    continum init
    ```
    
    This creates a configuration file and installs the pre-commit hook.
  </Step>
</Steps>

That's it! Every commit will now be scanned automatically.

## Verification

After installation, verify everything is working:

```bash
# Check CLI version
continum --version

# Check configuration status
continum status
```

You should see output confirming:
- ✓ Git repository detected
- ✓ Configuration file found
- ✓ Pre-commit hook installed
- ✓ Connected to Continum API

## What Gets Installed?

When you run `continum init`, the following files are created:

<CodeGroup>
```json .continum.json
{
  "scanOnCommit": true,
  "sandbox": "your_sandbox",
  "block": ["CRITICAL", "HIGH"],
  "warn": ["MEDIUM"],
  "ignore": [
    ".env.example",
    "**/*.test.ts"
  ]
}
```

```bash .git/hooks/pre-commit
#!/bin/sh
# Continum pre-commit hook
continum scan --staged --hook
```
</CodeGroup>

<Info>
  The configuration file is committed to your repository, but credentials are stored locally in `~/.continum/credentials.json`
</Info>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command not found: continum">
    The CLI wasn't installed globally. Run:
    ```bash
    npm install -g @continum/cli
    ```
  </Accordion>
  
  <Accordion title="Not logged in to Continum">
    You need to authenticate first:
    ```bash
    continum login
    ```
  </Accordion>
  
  <Accordion title="Not in a git repository">
    Continum requires a git repository. Initialize one:
    ```bash
    git init
    ```
  </Accordion>
  
  <Accordion title="Port 8765 already in use">
    Another process is using the authentication port. Stop it or wait a moment, then try again:
    ```bash
    continum login
    ```
  </Accordion>
</AccordionGroup>
