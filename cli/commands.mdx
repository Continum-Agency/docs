## Command Overview

The Continum CLI provides several commands for managing credential scanning and pattern detection.

## continum login

Authenticate with your Continum account.

```bash
continum login
```

**What it does:**
- Opens your browser for secure authentication
- Displays your API key in browser and terminal
- Saves credentials locally
- Tests connection to Continum

**You only need to run this once per machine.**

<Info>
  Your API key is displayed during login. Store it securely for your records.
</Info>

---

## continum init

Initialize Continum in a project.

```bash
continum init [options]
```

**Options:**
- `--silent` - Non-interactive mode (for automated scripts)

**What it does:**
- Creates `.continum.json` configuration file
- Installs pre-commit hook at `.git/hooks/pre-commit`
- Verifies connection to Continum API

**Requirements:**
- Must be in a git repository
- Must be logged in (run `continum login` first)

**Example:**
```bash
cd your-project
continum init
```

For automated deployment:
```bash
continum init --silent
```

## continum scan

Scan files for credentials and sensitive data.

```bash
continum scan [files...] [options]
```

**Arguments:**
- `[files...]` - Specific files to scan (optional)

**Options:**
- `--staged` - Scan staged files (used by pre-commit hook)
- `--hook` - Minimal output mode (for git hooks)
- `--strict` - Block on unknown patterns without prompting
- `--warn-only` - Show warnings but don't block commits

**Examples:**

Scan specific files:
```bash
continum scan src/config.ts src/utils.ts
```

Scan staged files (used by pre-commit hook):
```bash
continum scan --staged
```

Strict mode (block all unknown patterns):
```bash
continum scan --staged --strict
```

Warn-only mode (never block commits):
```bash
continum scan --staged --warn-only
```

## continum patterns

Manage credential patterns.

### Update Patterns

Update patterns from Continum API:

```bash
continum patterns update
```

This syncs the latest patterns from your team to your local machine.

### List Patterns

List all available patterns:

```bash
continum patterns list
```

Shows all patterns including:
- Built-in patterns (cloud providers, databases, etc.)
- Custom patterns approved by your team
- Pattern type and severity level

## continum status

Check Continum configuration and connection.

```bash
continum status
```

**Shows:**
- Git repository status
- Configuration file location
- Pre-commit hook status
- API connection status
- Your account information

**Example output:**
```bash
Continum Status
───────────────────────────────────────────────────

✓ Git repository detected
✓ Configuration file: .continum.json
✓ Pre-commit hook installed
✓ Connected to Continum API
✓ Account: Acme Corp (acme@example.com)

Everything looks good!
```

## continum uninstall

Remove Continum pre-commit hook from the repository.

```bash
continum uninstall
```

**What it does:**
- Removes the pre-commit hook
- Keeps the configuration file (you can delete it manually)
- Keeps your credentials (for use in other projects)

<Warning>
  This only removes the hook from the current repository. Your credentials remain for use in other projects.
</Warning>

## Global Options

These options work with any command:

- `--help` - Show help for a command
- `--version` - Show CLI version

**Examples:**
```bash
continum --help
continum scan --help
continum --version
```

---

## Exit Codes

The CLI uses standard exit codes:

- `0` - Success
- `1` - Error or violations found
- `2` - Invalid usage

These are useful for CI/CD integration:

```bash
continum scan --staged
if [ $? -eq 0 ]; then
  echo "Scan passed"
else
  echo "Scan failed"
  exit 1
fi
```
