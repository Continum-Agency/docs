## Overview

Continum CLI uses secure browser-based authentication to connect to your account. You only need to authenticate once per machine.

## Login Process

<Steps>
  <Step title="Run Login Command">
    Start the authentication process:
    
    ```bash
    continum login
    ```
  </Step>
  
  <Step title="Browser Opens">
    Your default browser opens to the Continum authentication page. If it doesn't open automatically, copy the URL from your terminal.
  </Step>
  
  <Step title="Sign In">
    Sign in with your Continum account credentials. If you don't have an account, you'll be prompted to create one.
  </Step>
  
  <Step title="API Key Display">
    After successful authentication, your API key is displayed in both:
    - The browser window (for 30 seconds)
    - Your terminal output
    
    <Warning>
      Copy and store your API key securely. While you can view it later in your dashboard, it's good practice to keep a backup.
    </Warning>
  </Step>
  
  <Step title="Automatic Configuration">
    The CLI automatically saves your credentials locally. You're now ready to use Continum.
  </Step>
</Steps>

## What Happens During Login

1. **Local Server Starts** - A temporary server runs on port 8765 to receive authentication
2. **Browser Authentication** - You authenticate securely through your browser
3. **Credentials Transfer** - Your API key is sent back to the CLI via localhost
4. **Secure Storage** - Credentials are saved to `~/.continum/credentials.json`
5. **Connection Test** - The CLI verifies it can connect to Continum

## Credentials Storage

Your credentials are stored locally in your home directory:

```bash
~/.continum/credentials.json
```

This file contains:
- Your API key
- The Continum API URL
- Your customer information

<Warning>
  Never commit this file to version control. It contains sensitive credentials.
</Warning>

## Security

<CardGroup cols={2}>
  <Card title="Browser-Based Auth" icon="shield-check">
    Authentication happens in your browser, not in the terminal. This ensures your credentials are never exposed in command history.
  </Card>
  
  <Card title="State Verification" icon="lock">
    A random state parameter prevents CSRF attacks during the authentication flow.
  </Card>
  
  <Card title="Local Storage" icon="folder-closed">
    Credentials are stored locally on your machine, never in your repository.
  </Card>
  
  <Card title="Encrypted API Keys" icon="key">
    API keys are encrypted at rest in the Continum platform.
  </Card>
</CardGroup>

## Managing API Keys

### View Your API Key

You can view your API key anytime in the Continum dashboard:

1. Go to [app.continum.co](https://app.continum.co)
2. Navigate to Settings → API Keys
3. Your key is displayed with copy functionality

### Rotate API Keys

If you need to rotate your API key:

1. Create a new API key in your dashboard
2. Run `continum login` again to update the CLI
3. Delete the old API key from your dashboard

### Multiple Machines

You need to authenticate on each machine where you use the CLI:

```bash
# On each new machine
continum login
```

Your credentials are machine-specific and not shared across devices.

## Environment Variables

For CI/CD environments, you can set credentials via environment variables:

```bash
export CONTINUM_API_KEY="your-api-key"
export CONTINUM_API_URL="https://api.continum.co"
```

The CLI will use these instead of the credentials file.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Browser doesn't open">
    Copy the URL from your terminal and paste it into your browser manually.
  </Accordion>
  
  <Accordion title="Authentication timeout">
    The authentication window expires after 5 minutes. Run `continum login` again to restart.
  </Accordion>
  
  <Accordion title="Port 8765 already in use">
    Another process is using the authentication port. Stop it or wait a moment, then try again.
  </Accordion>
  
  <Accordion title="Failed to connect to Continum API">
    Check your internet connection and verify the API is accessible. You can also check the status page.
  </Accordion>
  
  <Accordion title="No API key found">
    Your account doesn't have an API key yet. This is automatically created during signup. Contact support if you see this error.
  </Accordion>
</AccordionGroup>
